Chart.js Components for SolidJS

This directory contains Chart.js components built with solid-chartjs for
creating interactive charts in your SolidJS application.

Components

PerformanceChart

A simple chart component for displaying single-dataset performance metrics over
time.

Props:

  • title: string - Chart title
  • type: "line" | "bar" - Chart type
  • data: PerformanceDataPoint[] - Array of data points
  • width?: number - Chart width (default: 400)
  • height?: number - Chart height (default: 300)
  • yAxisLabel?: string - Y-axis label
  • showGrid?: boolean - Whether to show grid lines (default: true)
  • color?: string - Custom color for the chart

Example:

import { PerformanceChart } from '~/components/Charts';

const data = [
  { timestamp: 1000, value: 50, label: '10:00' },
  { timestamp: 2000, value: 75, label: '10:01' },
  { timestamp: 3000, value: 60, label: '10:02' },
];

<PerformanceChart
  title="Memory Usage Over Time"
  type="line"
  data={data}
  width={600}
  height={300}
  yAxisLabel="Memory (bytes)"
  color="rgba(255, 99, 132, 1)"
/>;

MultiDatasetChart

A more advanced chart component for displaying multiple datasets with different
chart types.

Props:

  • title: string - Chart title
  • type: "line" | "bar" | "doughnut" | "pie" - Chart type
  • labels: string[] - X-axis labels
  • datasets: Dataset[] - Array of datasets
  • width?: number - Chart width (default: 400)
  • height?: number - Chart height (default: 300)
  • xAxisLabel?: string - X-axis label
  • yAxisLabel?: string - Y-axis label
  • showGrid?: boolean - Whether to show grid lines (default: true)
  • showLegend?: boolean - Whether to show legend (default: true)
  • colors?: string[] - Custom colors for datasets

Example:

import { MultiDatasetChart } from '~/components/Charts';

const labels = ['Jan', 'Feb', 'Mar', 'Apr'];
const datasets = [
  {
    label: 'Sales',
    data: [10, 20, 30, 40],
  },
  {
    label: 'Revenue',
    data: [15, 25, 35, 45],
  },
];

<MultiDatasetChart
  title="Sales vs Revenue"
  type="bar"
  labels={labels}
  datasets={datasets}
  width={600}
  height={300}
  xAxisLabel="Month"
  yAxisLabel="Amount"
  showLegend={true}
  colors={['rgba(255, 99, 132, 1)', 'rgba(75, 192, 192, 1)']}
/>;

Installation

The required dependencies are already installed in your project:

{
  "chart.js": "^4.5.0",
  "solid-chartjs": "1.3.11"
}

Usage in PerformanceDashboard

The PerformanceDashboard component now includes real-time charts that display:

  1. Memory Usage Over Time - Shows memory consumption trends
  2. Browser Responsiveness - Displays browser response times
  3. Frame Rate - Shows FPS over time
  4. Selection Duration - Displays selection operation performance

The charts automatically update every second when the dashboard is visible and
recording is active.

Features

  • Real-time Updates: Charts automatically update with new data
  • Responsive Design: Charts adapt to different screen sizes
  • Theme Integration: Charts use CSS custom properties for theming
  • Interactive Tooltips: Hover over data points to see detailed information
  • Multiple Chart Types: Support for line, bar, doughnut, and pie charts
  • Customizable Colors: Apply custom colors to match your design
  • Grid Lines: Optional grid lines for better readability
  • Legends: Configurable legends for multi-dataset charts

Best Practices

  1. Register Chart.js Components: The components automatically register
    required Chart.js elements on mount
  2. Data Format: Ensure your data follows the expected format with timestamps
    and labels
  3. Performance: For real-time charts, consider limiting the number of data
    points to prevent memory issues
  4. Responsive Design: Use appropriate width and height values for different
    screen sizes
  5. Color Consistency: Use consistent colors across related charts for better
    UX

Testing

Both components include comprehensive test suites:

npm test -- --run src/components/Charts/

The tests cover:

  • Component rendering with different props
  • Chart type switching
  • Custom color application
  • Empty data handling
  • Multiple dataset support
