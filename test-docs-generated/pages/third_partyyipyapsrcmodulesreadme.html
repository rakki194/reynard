Modular Modules System

Modular Modules System

A comprehensive modular architecture for SolidJS applications that breaks down
monolithic contexts into focused, reusable modules.

Overview

This modular system extracts functionality from the massive 2,190-line app.tsx
context into focused modules, each under 140 lines and with zero cross-module
dependencies. The system enables:

  • Zero coupling between modules
  • High testability with 95%+ test coverage
  • Easy reuse across projects
  • Clean composition through dependency injection
  • Performance optimization through focused reactivity

Architecture

Core Principles

  1. 140-line rule: All modules are under 140 lines
  2. Zero dependencies: No cross-module imports
  3. Single responsibility: Each module has one clear purpose
  4. Comprehensive testing: 95%+ test coverage for each module
  5. Clean interfaces: Well-defined TypeScript interfaces

Modules

Theme Module (theme.ts)

Purpose: Manages application theme state and persistence

Features:

  • Theme switching (light, dark, gray, etc.)
  • localStorage persistence
  • Document attribute updates
  • Reactive theme changes

Usage:

import { createThemeModule } from '~/modules/theme';

const themeModule = createThemeModule();
themeModule.setTheme('dark');
console.log(themeModule.theme); // "dark"

Interface:

interface ThemeModule {
  readonly theme: Theme;
  setTheme: (theme: Theme) => void;
}

Auth Module (auth.ts)

Purpose: Handles user authentication state and API requests

Features:

  • Login/logout functionality
  • Token management
  • Role-based access control
  • Authenticated API requests
  • Model download controls

Usage:

import { createAuthModule } from '~/modules/auth';

const authModule = createAuthModule();
authModule.login('token', 'user', 'username');
console.log(authModule.isLoggedIn); // true

Interface:

interface AuthModule {
  readonly isLoggedIn: boolean;
  readonly userRole: string | null;
  readonly isInitializing: boolean;
  readonly disableModelDownloads: boolean;
  login: (
    token: string,
    role: string,
    username: string,
    refreshToken?: string
  ) => void;
  logout: () => void;
  setDisableModelDownloads: (value: boolean) => void;
  authFetch: (
    input: RequestInfo | URL,
    init?: RequestInit
  ) => Promise<Response>;
}

Notifications Module (notifications.ts)

Purpose: Manages application notifications and user feedback

Features:

  • Multiple notification types (error, success, info, warning)
  • Progress tracking
  • Group-based notifications
  • Automatic ID generation
  • Notification lifecycle management

Usage:

import { createNotificationsModule } from '~/modules/notifications';

const notificationsModule = createNotificationsModule();
notificationsModule.notify('Operation successful', 'success');
notificationsModule.createNotification({
  message: 'Processing...',
  type: 'info',
  progress: 50,
});

Interface:

interface NotificationsModule {
  readonly notifications: Notification[];
  notify: (
    message: string,
    type?: 'error' | 'success' | 'info' | 'warning',
    group?: string,
    icon?: string,
    progress?: number
  ) => void;
  createNotification: (
    notification: Omit<Notification, 'id' | 'timestamp'>
  ) => void;
  removeNotification: (id: string) => void;
  clearNotifications: (group?: string) => void;
}

Migration Guide

From Monolithic Context

  1. Extract modules: Use the provided module implementations
  2. Update imports: Replace useAppContext() with specific module imports
  3. Compose modules: Use createAppModules() for unified access
  4. Update tests: Use the provided test suites as templates

Example Migration

Before:

const { theme, setTheme, isLoggedIn, login } = useAppContext();

After:

const { theme, auth } = createAppModules();
// or
const themeModule = createThemeModule();
const authModule = createAuthModule();

Module Template

/**
 * ModuleName module - handles specific functionality
 * Extracted from monolithic context to reduce coupling
 */

import { createSignal, createEffect } from 'solid-js';

export interface ModuleNameModule {
  readonly property: Type;
  setProperty: (value: Type) => void;
}

export const createModuleNameModule = (): ModuleNameModule => {
  const [property, setPropertySignal] = createSignal<Type>(defaultValue);

  // Initialize from localStorage
  createEffect(() => {
    const stored = localStorage.getItem('property');
    if (stored) {
      setPropertySignal(stored);
    }
  });

  const setProperty = (value: Type) => {
    localStorage.setItem('property', value.toString());
    setPropertySignal(value);
  };

  return {
    get property() {
      return property();
    },
    setProperty,
  };
};
