Training Scripts Organization

This directory contains training scripts with a simple organization structure that loads existing scripts directly and allows users to create new scripts in their own subdirectories.

Directory Structure

training_scripts/
├── README.md                    # This file
├── *.py                        # Existing training scripts (loaded directly)
├── user/                       # User-created scripts
│   └── [username]/             # Scripts organized by user
│       ├── [script_name].py    # User scripts with clean filenames
│       └── [directory_name]/   # User-created directories
└── [other files]               # Any other files in the directory

How It Works

Existing Scripts

  • All Python files (.py) in the root training_scripts/ directory are automatically loaded into the editor
  • These scripts are treated as "built-in" scripts available to all users
  • Scripts are loaded with their original filenames and content
  • No migration or UUID system is used

User Scripts

  • Users can create new scripts in their own subdirectory: training_scripts/user/[username]/
  • Each user has their own isolated workspace
  • Script names are automatically converted to clean filenames (spaces become underscores, lowercase)
  • User scripts are stored with descriptive names, not UUIDs

Script Management

Creating Scripts

  • Use the web interface in the Training Script Editor
  • Scripts are automatically saved to the user's subdirectory
  • Filenames are cleaned automatically (e.g., "My Test Script" becomes "my_test_script.py")

Editing Scripts

  • Existing scripts can be edited directly
  • Changes are saved back to the original files
  • User scripts are saved to their respective user directories

Deleting Scripts

  • Scripts can be deleted through the web interface
  • Deletion removes the actual files from the filesystem
  • Use with caution as this is permanent

API Endpoints

The training script system provides the following API endpoints:

  • GET /api/training/scripts/tree - Get all scripts (existing + user scripts)
  • POST /api/training/scripts - Create a new script
  • GET /api/training/scripts/{id} - Get a specific script
  • PUT /api/training/scripts/{id} - Update a script
  • DELETE /api/training/scripts/{id} - Delete a script

Benefits of This Organization

  1. Simple and Direct: No complex migration or UUID systems
  2. User Isolation: Each user's scripts are kept separate
  3. Clean Filenames: User scripts get automatically cleaned filenames
  4. Direct File Access: Scripts are stored as regular Python files
  5. Easy Backup: Simple directory structure is easy to backup and version control
  6. No Database Dependencies: Everything is file-based
  7. Immediate Availability: Existing scripts are loaded instantly

Adding New Scripts

To add a new script:

  1. Via Web Interface: Use the Training Script Editor in the application
  2. Direct File Creation: Create Python files directly in the appropriate directory
    • For built-in scripts: Add to training_scripts/
    • For user scripts: Add to training_scripts/user/[username]/

Best Practices

  1. Use Descriptive Names: Script names should clearly indicate their purpose
  2. Add Comments: Include docstrings and comments explaining the script's functionality
  3. Keep User Scripts Organized: Use subdirectories within your user folder for better organization
  4. Backup Regularly: The simple file structure makes it easy to backup your scripts
  5. Test Scripts: Always test your scripts before running them in production

File Naming

  • Existing Scripts: Keep their original names (e.g., tslearn_demo.py)
  • User Scripts: Will be automatically converted to clean names:
    • "My Training Script" → my_training_script.py
    • "Aesthetic Scorer" → aesthetic_scorer.py
    • "Test Script 123" → test_script_123.py

Security

  • User scripts are isolated in their own directories
  • No cross-user access to scripts
  • File permissions are maintained as set by the filesystem
  • Script execution is controlled by the application's security model
